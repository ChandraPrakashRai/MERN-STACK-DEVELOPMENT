Lecture 7 — Video

Theory

1. Why Media Tags? Beyond Text & Images

Internet aur web pages ke pahle versions sirf text aur images hi display karte the. Agar video ya audio dikhani hoti thi, toh plugins (jaise Flash, Silverlight) use karne padte the — jo slow, insecure, aur incompatible hote the.

HTML5 ne ye problem solve kiya by introducing native media tags: <video> and <audio>. Inse browser pe bina kisi extra plugin ke media play kiya ja sakta hai.

2. Basic Usage of <video>

  • src attribute: video file ka path (local ya online).
  • controls: agar ye attribute hai, toh video player controls (play/pause, volume, seek bar) dikhte hain.
  • width & height: size specify kar sakte ho.
<video src="sample-video.mp4" controls width="640" height="360"></video>

3. Advanced Video Tag Attributes

Niche kuch attributes hain jo video ko zyada functional banate hain:

  • autoplay: Page load hote hi video start karega. (Lekin sound ke saath autoplay many browsers block karte hain.)
  • muted: Video bina sound ke. Autoplay ke saath combined use hota hai.
  • loop: Video end hone ke baad automatically dobara start karega.
  • poster: Ek image dikhata hai jab video load nahi hua ho ya play nahi hua ho, jaise thumbnail.
<video src="promo.mp4" controls autoplay muted loop poster="images/promo-thumbnail.jpg" width="640" height="360">
</video>

4. Providing Multiple Formats for Compatibility

Har browser sab video format nahi support karta. Ek format chalta hai Chrome me, lekin same format Safari ya mobile par na chale toh problem hogi.

Isliye <source> tag use karke multiple formats provide karte hain (jaise .mp4, .webm, .ogg) aur browser pehle supported format play karega.

<video controls poster="images/video-thumb.jpg">
  <source src="video.webm" type="video/webm">
  <source src="video.mp4" type="video/mp4">
  Sorry, your browser does not support embedded videos.
</video>

5. The <audio> Tag

Audio tag lagbhag video jaise hi behave karta hai, bas visual component nahi hota.

  • src ka path dena padta hai.
  • controls attribute audio player controls ko enable karta hai.
  • Autoplay, loop etc attributes bhi common hain.
<audio src="audio/song.mp3" controls></audio>

6. Accessibility: Subtitles, Captions, & Tracks

Sabhi users ek jaise nahi sun sakte ya dekh sakte hain. Kuch log deaf ya hard of hearing hote hain; kuch noisy environment ya background noise me video dekhte hain. <track> tag se captions/subtitles add ki ja sakti hain. 

<video controls>
  <source src="video.mp4" type="video/mp4">
  <track src="captions_en.vtt" kind="captions" srclang="en" label="English">
</video>

7. Fallback Text & Browser Support

Agar browser video tag nahi samajhta (bahut purana browser), fallback text show karo jaise: "Your browser does not support embedded videos." Ye user ko batata hai ki wo video nahi dekh pa rahe hain, aur option ho sakta hai ki download link provide karo. 

<video controls>
  <source src="video.mp4" type="video/mp4">
  <source src="video.ogg" type="video/ogg">
  Your browser does not support embedded videos.
</video>

Summary

  • HTML5 media tags (<video> & <audio>) ne media embedding ko plugin-free banaya.
  • Use controls, autoplay, muted, loop, poster as needed.
  • Multiple source formats ensure compatibility across devices/browsers.
  • Accessibility ke liye captions/subtitles <track> ka use karo.
  • Fallback text zaroori hai old browsers ke liye.
