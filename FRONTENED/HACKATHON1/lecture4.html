Lecture 4: File Paths & HTML Boilerplate

Video Lecture

1. File Path System

Websites are built from multiple files like HTML, CSS, JavaScript, and images. These files need to be well-organized in folders.

Relative File Paths

These are based on the location of the current file.

my-website/
├── index.html
├── about.html
├── images/
│   ├── logo.png
│   └── banner.jpg
└── pages/
    ├── contact.html
    └── terms.html

Examples:

  • Same folder: <a href="about.html">About</a>
  • Sub-folder: <img src="images/logo.png" alt="Logo">
  • Parent folder: <img src="../images/logo.png" alt="Logo">

Absolute File Paths

These are full web addresses (URLs) for resources outside your site.

<a href="https://www.google.com">Google</a>
<img src="https://upload.wikimedia.org/wikipedia/commons/6/6a/Html5_logo_and_wordmark.svg" alt="HTML5 Logo">

2. HTML Boilerplate

The boilerplate is the basic starting template for every HTML page.

<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Document</title>
</head>
<body>

</body>
</html>

Explanation:

  • <!DOCTYPE html>: Use HTML5
  • <html lang="en">: Language of the page
  • <meta charset="UTF-8">: Supports all characters
  • <meta name="viewport">: Makes site mobile-friendly
  • <title>: Shown in browser tab

3. Multi-Page Websites

Most websites have multiple pages like Home, About, Contact. You can connect them using navigation links.

<nav>
    <a href="index.html">Home</a>
    <a href="about.html">About</a>
    <a href="pages/contact.html">Contact</a>
</nav>

4. Containers: <div> & <span>

<div> Example

<div>
    <img src="PLACEHOLDER_PROFILE_IMAGE" alt="Profile Picture">
    <h2>John Doe</h2>
    <p>Web Developer</p>
</div>

Class and ID

<div id="main-header">Header</div>
<div class="profile">Profile 1</div>
<div class="profile">Profile 2</div>

<span> Example

<p>This product is <span>50% OFF</span> today only!</p>

5. Semantic Containers

  • <header>: Top section (logo, nav)
  • <footer>: Bottom section (contact info)
  • <main>: Main content
