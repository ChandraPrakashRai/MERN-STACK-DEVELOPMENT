Notes

Principle 1: The Need to Share Information

  • Information exists on one computer but needs to reach another.
  • Direct physical transfer (USB) is slow and non-scalable.
  • Using cables allows electrical or light signals to transmit data.
  • This creates a basic network: a direct link between two computers.

Principle 2: The Problem of Scale

  • Direct links between multiple computers require many cables (scales poorly).
  • Solution: Connect computers to a central switch → forms a local network.

Principle 3: Connecting Networks

  • To connect multiple networks, we need routers.
  • Routers forward messages between networks efficiently.
  • The Internet is a "network of networks".

Principle 4: Addressing

  • Every device needs a unique address (IP address).
  • Routers forward packets based on destination IP address.

Principle 5: Reliability and Fairness

  • Large files are broken into packets for efficient, reliable transfer.
  • TCP/IP protocol ensures correct ordering and error handling.

IP Address

  • IPv4: 32-bit address, example: 172.217.16.142
  • Public IP: Globally unique, assigned by ISP
  • Private IP: Local network only, assigned by router

IPv6

  • 128-bit address to solve IPv4 exhaustion
  • Example: 2001:0db8:85a3:0000:0000:8a2e:0370:7334

MAC Address

  • 48-bit unique hardware identifier
  • Formats: Colon-separated, Hyphen-separated, Period-separated

Port Numbers

  • Identify specific applications/services on a device
  • Range 0-65535: Well-known (0-1023), Registered (1024-49151), Dynamic/Private (49152-65535)

DNS (Domain Name System)

  • Translates human-friendly domain names to IP addresses
  • Uses local cache, recursive resolver, and hierarchical search to find IP
  • Final connection opens direct communication with destination server
